课程：编译原理

一、语法分析概述

语法分析的功能

语法分析是编译程序的核心部分，其任务是检查词法分析器输出的单词序列是否是源语言中的句子，即是否符合源语言的语法规则。

词法分析器 -> 语法分析器 -> 前端的其它部分（语义分析、中间代码生成）
↑
符号表

两种分析方式

方式 方向 本质 典型方法
自顶向下 从开始符号推导出句子 产生语言 递归下降、预测分析法(LL(1))
自底向上 将单词序列归约为开始符号 识别语言 算符优先、LR分析法

二、自顶向下分析面临的问题

自顶向下分析是一种试探性过程，可能导致分析效率极低甚至失败。主要面临三个问题：

1. 二义性问题

定义：如果文法G的某个句子存在两棵或两棵以上不同的语法分析树，则称G是二义性的。

示例文法：

E -> id | c | E + E | E - E | E * E | E / E | (E)

二义性句子：id + c * id 存在两棵不同的语法树（乘加优先级未定义）

解决方法：

  • 改造文法，引入新的语法变量明确优先级
  • 或根据优先级关系保证高优先级运算符优先

2. 左递归引起的无穷推导问题

定义：如果存在推导 A =>+ Aα，则称文法G是递归的。当 α=ε 时称为左递归。

示例：

E -> E + T | E - T | T
T -> T * F | T / F | F
F -> (E) | id

问题：为输入串 id+id*id 建立最左推导时会陷入无限循环。

解决方法：消除左递归

3. 回溯问题

定义：如果非终结符A有多个候选式存在公共前缀，则自顶向下分析程序无法根据当前输入符号准确选择产生式，只能试探，试探失败时需要回退。

示例：

-> if then
| if then else

两个候选式都以 if then 开头，需要试探。

解决方法：提取左因子

三、文法的改造

3.1 消除左递归

（1）直接左递归的消除

将左递归产生式 A -> Aα | β 替换为：

A -> β A’
A’ -> α A’ | ε

示例：

原产生式：
E -> E + T | T

消除后：
E -> T E’
E’ -> + T E’ | ε

（2）间接左递归的消除

算法（按编号代入法）：

  1. 将语法变量排序：A1, A2, …, An
  2. for i = 1 to n:
    • for j = 1 to i-1:
      • 将每个形如 Ai -> Aj β 的产生式替换为 Ai -> α1 β | α2 β | … | αk β
        （其中 Aj -> α1 | α2 | … | αk）
    • 消除 Ai 的直接左递归

示例：消除下列文法的左递归

S -> A c | c
A -> B b | b
B -> S a | a

步骤：

  1. 排序：B, A, S
  2. i=1（B）：无直接左递归
  3. i=2（A）：代入 B -> S a | a，得 A -> S a b | a b | b
  4. i=3（S）：代入 A，得 S -> S a b c | a b c | b c | c，消除直接左递归

3.2 提取左因子

方法：将公共前缀 α 提取出来

原产生式：
A -> α β1 | α β2 | … | α βn | γ1 | γ2 | … | γm

提取后：
A -> α A’ | γ1 | γ2 | … | γm
A’ -> β1 | β2 | … | βn

示例：

原产生式：
-> if then
| if then else

提取后：
-> if then
-> ε | else

四、LL(1)文法

4.1 FIRST集

定义：设α是文法符号串，FIRST(α)是从α可以推导出的所有串的首终结符集合。若 α =>* ε，则 ε ∈ FIRST(α)。

计算算法：

  1. 若 X ∈ T，则 FIRST(X) = {X}
  2. 若 X ∈ V，且 X -> ε ∈ P，则 ε ∈ FIRST(X)
  3. 若 X ∈ V，且 X -> Y1 Y2 … Yk ∈ P，则：
    • 将 FIRST(Y1) - {ε} 加入 FIRST(X)
    • 若 Y1 =>* ε，则将 FIRST(Y2) - {ε} 加入，依此类推
    • 若所有 Yj =>* ε，则 ε ∈ FIRST(X)

示例：表达式文法

E -> T E’
E’ -> + T E’ | ε
T -> F T’
T’ -> * F T’ | ε
F -> (E) | id

FIRST集计算结果：

符号 FIRST集
F {(, id}
T {(, id}
E {(, id}
E’ {+, ε}
T’ {*, ε}

4.2 FOLLOW集

定义：FOLLOW(A)是所有句型中紧跟在A后面的终结符集合。若A是某个句型的最右符号，则 # ∈ FOLLOW(A)。

计算算法：

  1. 将 # 放入 FOLLOW(S)
  2. 若有产生式 A -> α B β，则将 FIRST(β) - {ε} 加入 FOLLOW(B)
  3. 若有产生式 A -> α B 或 A -> α B β 且 β =>* ε，则将 FOLLOW(A) 加入 FOLLOW(B)

示例：表达式文法的FOLLOW集

符号 FOLLOW集
E {), #}
E’ {), #}
T {+, ), #}
T’ {+, ), #}
F {*, +, ), #}

4.3 LL(1)文法的判定条件

对于文法G的任意两个具有相同左部的产生式 A -> α | β：

  1. 若α、β都不能推导出ε，则 FIRST(α) ∩ FIRST(β) = ∅
  2. α和β至多有一个能推导出ε
  3. 若 β =>* ε，则 FIRST(α) ∩ FOLLOW(A) = ∅

满足上述条件的文法称为LL(1)文法。

  • 第一个L：从左到右扫描输入
  • 第二个L：产生最左推导
  • 1：每步向前查看一个输入符号

五、预测分析法

5.1 预测分析器的构成

+—————–+
| 输入缓冲区 | <- 待分析的符号串 + #
+—————–+
|
+—————–+
| 栈 | <- 文法符号序列，栈底为#
+—————–+
| 预测分析程序 |
+—————–+
| 预测分析表 M |
+—————–+
|
+—————–+
| 输出的产生式 |
+—————–+

5.2 预测分析表的构造算法

输入：文法G
输出：预测分析表M

步骤：

  1. 对每个产生式 A -> β：
    • 对 ∀a ∈ FIRST(β)，将 A -> β 填入 M[A, a]
    • 若 ε ∈ FIRST(β)，则对 ∀a ∈ FOLLOW(A)，将 A -> β 填入 M[A, a]
  2. 所有未定义的表项标记为 error

5.3 预测分析程序的总控算法

  1. 将栈底符号#和文法开始符号S压入栈中
  2. repeat
  3. X := 当前栈顶符号; a := 当前输入符号
  4. if X ∈ T ∪ {#} then
  5. if X = a then

  6.   if X ≠ # then 将X弹出栈; 前移输入指针

  7.   if X = # then break  // 分析成功停机

  8. else 转向出错处理程序
  9. if X ∈ V then
  10. if M[X, a] = Y1 Y2 … Yk then
  11.  将X弹出栈

  12.  依次将Yk, ..., Y2, Y1压入栈

  13.  输出产生式 X -> Y1 Y2 ... Yk
  14. else 转向出错处理程序
  15. until false

5.4 分析示例

文法：

E -> T E’
E’ -> + T E’ | ε
T -> F T’
T’ -> * F T’ | ε
F -> (E) | id

分析表：

+——–+—–+—+—+—–+—+—+
| 非终结符 | id | + | * | ( | ) | # |
+——–+—–+—+—+—–+—+—+

| E’ | | E’->+TE’ | | | E’->ε | E’->ε |

六、递归下降分析法

基本思想

为文法的每个非终结符编写一个处理程序（子程序），用来识别该非终结符所代表的语法成分。

示例代码

void E() {
T();
E’();
}

void E’() {
if (lookahead == ‘+’) {
match(‘+’);
T();
E’();
}
// else ε
}

void T() {
F();
T’();
}

void T’() {
if (lookahead == ‘‘) {
match(‘‘);
F();
T’();
}
// else ε
}

void F() {
if (lookahead == ‘id’)
match(‘id’);
else if (lookahead == ‘(‘) {
match(‘(‘);
E();
match(‘)’);
}
else error();
}

优缺点

优点 缺点
直观、简单、可读性好 递归算法效率低
便于扩充 处理能力相对有限
通用性差，难以自动生成

七、错误处理

紧急方式恢复策略

  1. 跳过输入符号直到遇到同步记号
  2. 同步记号的选择：
    • 将 FOLLOW(A) 的符号放入 A 的同步记号集合
    • 将高层结构的开始符号加入低层结构的同步记号
    • 将 FIRST(A) 的符号加入 A 的同步记号

错误处理示例

在预测分析表中，将同步记号对应的表项标记为 “synch”：

  • 若 M[A, a] 为空 -> 跳过输入符号 a
  • 若 M[A, a] = synch -> 弹出栈顶的 A
  • 若栈顶终结符与输入不匹配 -> 弹出该终结符
