课程：编译原理

一、词法分析器的功能

1.1 基本功能

  • 输入：源程序字符串（字符流）
  • 输出：单词符号序列（种别码，属性值）

1.2 主要任务

任务 说明
识别单词 根据词法规则组合单词
词法检查 发现非法字符、拼写错误等
常数转换 数字字符串 → 二进制数值
预处理 删除空格、注释等无关字符

1.3 单词的种类

类型 示例 特点
关键字 begin, end, for, do 预定义，数量固定
标识符 变量名、函数名 用户定义，数量不定
常数 整常数、实常数、字符串 数量不定
运算符 +, -, *, /, >, <, = 预定义
分界符 ; , ( ) { } 预定义

1.4 单词的内部表示

二元组形式：(种别码, 属性值)

单词类别 种别码 属性值
标识符 1 指向符号表入口的指针
无符号整数 2 整数值
关键字 6 内部编码
运算符 8 内部编码

示例：if count > 7 then result := 3.14;

(IF, 0)
(ID, 指向count的符号表入口)
(GT, 0)
(INT, 7)
(THEN, 0)
(ID, 指向result的符号表入口)
(ASSIGN, 0)
(REAL, 3.14)
(SEMIC, 0)

1.5 输入缓冲与预处理

超前搜索与回退

  • 识别双字符运算符（如 <=, <>, **）需要超前搜索
  • 回退操作修正超前搜索的影响

双缓冲区技术

┌─────────────────┬─────────────────┐
│ 缓冲区1 (N) │ 缓冲区2 (N) │
└─────────────────┴─────────────────┘
↑
lexeme_beginning
↑
forward

  • 避免磁盘频繁访问
  • 可识别单词长度扩展到 N
  • 使用 EOF 标记减少边界判断次数

1.6 错误处理

错误类型 处理方式
非法字符 检查合法字符集合
单词拼写错误 部分留待语法分析阶段检测
注释/字符串不封闭 限制长度或报错
重复声明 查填符号表时检测

错误恢复：紧急方式恢复（panic-mode）—— 删除字符直到能识别正确单词

二、单词的描述

2.1 正则文法（3型文法）

标识符的文法：

→ | |
→ A | B | … | Z | a | b | … | z
→ 0 | 1 | 2 | … | 9

2.2 正则表达式

基本运算

运算 符号 含义 示例
或 | 并集 a|b = {a, b}
连接 并列 连接 ab = {ab}
克林闭包 * 零次或多次 a* = {ε, a, aa, aaa, …}
正闭包 + 一次或多次 a+ = {a, aa, aaa, …}
可选 ? 零次或一次 a? = {ε, a}

优先级

* > 连接 > |

标识符的正则表达式

letter(letter|digit)*

常用示例

正则表达式 语言
a|b {a, b}
(a|b)(a|b) {aa, ab, ba, bb}
(a|b)* a和b组成的任意串（含ε）
a|a*b {a, b, ab, aab, aaab, …}

2.3 正则文法与正则表达式的等价性

正则文法 → 正则表达式

  1. 为每个产生式构造方程式
  2. 代入消元法求解
  3. 得到开始符号 S 的解

正则表达式 → 正则文法

分解规则：

形式 转换
A → r₁r₂ A → r₁B, B → r₂
A → r₁*r₂ A → r₁A, A → r₂
A → r₁|r₂ A → r₁, A → r₂

2.4 有穷状态自动机

DFA（确定的有穷自动机）

定义：M = (Q, Σ, δ, q₀, F)

符号 含义
Q 有穷状态集合
Σ 输入字母表
δ 转移函数：Q × Σ → Q
q₀ 初始状态
F 终止状态集合

DFA 的三种表示

  1. 转移函数：δ(0, a) = 1
  2. 转移矩阵：
状态 a b
0 1 2
1 3 2
2 1 3
3 3 3
  1. 状态转换图：顶点=状态，边=转移，双层圈=终止状态

NFA（非确定的有穷自动机）

  • δ 是 多值映射：Q × (Σ ∪ {ε}) → 2^Q
  • 同一状态读同一字符可进入多个状态
  • NFA 与 DFA 等价（每个 NFA 可转换为等价的 DFA）

2.5 状态转换图

构造规则：

  1. 每个语法变量对应一个状态结点
  2. 开始符号对应初始状态
  3. 增设终止状态 T
  4. A → aB：状态 A →(a)→ 状态 B
  5. A → a：状态 A →(a)→ 终止状态 T
  6. A → ε：状态 A →(any)→ 终止状态 T

2.6 正则表达式 → 状态转换图

基本转换规则：

正则表达式 状态转换图
∅ 无边
ε 开始→终止（ε边）
a 开始→(a)→终止
r|s 并联结构
rs 串联结构
r* 自环+ε边
r+ 自环（至少一次）

三、单词的识别

3.1 识别过程

  1. 从初始状态出发
  2. 读入一个字符
  3. 按当前字符转入下一状态
  4. 重复直到无法继续
  5. 若在终止状态，识别成功

3.2 典型单词识别

标识符与关键字识别

  • 关键字可作保留字（禁止作为标识符）
  • 识别策略：先按标识符识别，再查关键字表

常数识别

进制 正则表达式
八进制 0(0-7)(0-7)*
十进制 (1-9)(0-9)*|0
十六进制 0x(0-9a-f)(0-9a-f)*

关系运算符识别

< → LT
<= → LE
<> → NE
= → EQ

→ GT
= → GE

3.3 状态转换图的实现

方法1：状态矩阵

  • 行 = 状态，列 = 输入符号
  • 优点：查询速度快
  • 缺点：空间浪费大

方法2：邻接表

  • 每个状态维护一个链表
  • 优点：节省空间
  • 缺点：访问速度较慢

方法3：压缩数组（四数组结构）

  • base[s]：转移向量起始位置
  • default[s]：默认转向
  • next：目标状态
  • check：验证当前状态

查找算法：
if (check[base[s] + a] == s)
return next[base[s] + a]
else
进入默认转向

3.4 词法分析程序实现（伪代码）

token token_scan() {
    char ch;
    
    // 跳过空白字符
    ch = getchar();
    while (ch == ' ' || ch == '\t' || ch == '\n') {
        ch = getchar();
        lexeme_beginning++;
    }
    
    // 标识符识别
    if (isalpha(ch)) {
        ch = getchar();
        while (isalnum(ch))
            ch = getchar();
        retract(1);
        token = copytoken();
        return (gettoken(token), install_id(token));
    }
    
    // 整数识别
    if (isdigit(ch)) {
        ch = getchar();
        while (isdigit(ch))
            ch = getchar();
        retract(1);
        token = copytoken();
        return (INT, install_num(token));
    }
    
    // 运算符和分界符
    switch (ch) {
        case '*': 
            ch = getchar();
            if (ch == '*') return (EXP, 0);
            retract(1); return (MULTI, 0);
        case ':':
            ch = getchar();
            if (ch == '=') return (ASSIGN, 0);
            retract(1); return (COLON, 0);
        case '<':
            ch = getchar();
            if (ch == '=') return (LE, 0);
            if (ch == '>') return (NE, 0);
            retract(1); return (LT, 0);
        // ... 其他情况
        default:
            error_handle();
    }
}
